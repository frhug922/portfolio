### ✅ 구현 내용

### 1) JSON 기반 테이블 구조 설계 및 로드 시스템

- 기능 설명: 미연시/비주얼 노벨의 대화 내용 및 선택지 등 다양한 데이터를 JSON 테이블로 관리하고, 런타임 시 이를 파싱하여 각 시스템에서 불러오도록 할 수 있는 데이터 로드 시스템입니다.
- 구현 내용: TableManager 를 중심으로 TDialogs, TChoices 등 테이블들을 따로 파일로서 관리하고, 해당 파일을 교체하면 코드 수정 없이 즉시 반영되도록 구현했습니다.
- 선택 이유: ScriptableObject나 CSV기반 관리보다 수정 및 배포 과정이 단순하며, Google Sheet 등 외부 데이터 관리 도구와의 연동이 수월하기 때문입니다.
- 결과: 코드를 수정할 필요 없이 테이블 파일만 교체하면 게임 시스템과 독립적으로 스토리가 변경되도록 구현하였습니다.
이를 통해 추후 스토리를 추가하거나 교체할 때 유지보수가 한층 용이해졌습니다.

### 2) 데이터 저장/불러오기 시스템

- 기능 설명: 게임 진행 상황, 설정 등 게임에 필요한 데이터를 로컬에 저장하고 불러오는 시스템입니다.
- 구현 내용: PlayerPrefs를 활용하여 간단한 설정 데이터부터, JSON 직렬화를 통해 복잡한 게임 상태 데이터까지 저장하고 불러올 수 있도록 구현했습니다.
- 선택 이유: PlayerPrefs는 Unity에서 기본적으로 제공하는 간단한 데이터 저장 방법으로, 빠른 구현이 가능하며, JSON 직렬화를 통해 확장성을 확보할 수 있기 때문입니다. 그리고 로컬 저장소를 사용함으로써 네트워크 연결이 불필요한 환경에서도 안정적으로 동작할 수 있습니다.
- 결과: 게임 진행 상황과 설정을 저장하고 불러오는 기능이 안정적으로 동작하여, 플레이어가 게임을 중단했다가 다시 시작할 때 이전 상태를 유지할 수 있게 되었습니다.
