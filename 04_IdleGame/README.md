### ✅ 구현 내용

### 1) Utility 시스템 - BigCurrency 재화 도입

- 기능 설명: 게임 내 재화를 일반 정수형이 아닌 BigCurrency 타입으로 관리하여, 매우 큰 수치의 재화도 손실 없이 처리할 수 있는 시스템입니다.
- 구현 내용: BigCurrency 구조체를 설계하여, value와 tier를 함께 관리. ToString 등 기본 연산자 오버로딩을 통해 일반 정수형처럼 사용할 수 있도록 구현했습니다.
- 선택 이유: 방치형 게임에서는 재화가 기하급수적으로 증가하는 경우가 많아, 일반 정수형으로는 표현할 수 없는 큰 수치를 다뤄야 했습니다. BigCurrency 타입을 도입함으로써 이러한 문제를 해결하고, 코드의 가독성과 유지보수성을 높일 수 있었습니다.
- 결과: 모든 재화 관련 시스템에서 BigCurrency 타입을 사용하여, 매우 큰 수치의 재화도 안정적으로 처리할 수 있게 되었고, 게임의 확장성도 향상되었습니다.

### 2) FireBase 기반 서버 연동 시스템

- 기능 설명: Firebase Realtime Database를 활용하여 게임 데이터를 클라우드에 저장하고 불러오는 시스템입니다.
- 구현 내용: 프로필, 인벤토리, 재화 등 주요 게임 데이터를 Firebase와 연동하여 저장 및 불러오기 기능을 구현했습니다. 또한, 데이터 동기화 및 오류 처리 로직도 포함시켰습니다.
- 선택 이유: 별도의 서버 구축 없이도 안정적인 데이터 저장 및 관리를 할 수 있는 Firebase를 선택했습니다. 또한, 실시간 데이터 동기화 기능이 비정상 종료 후에도 플레이어의 진행 상황을 안전하게 보존할 수 있도록 도와주었습니다.
- 결과: 데이터 저장이 실시간으로 이루어져, 플레이어가 언제 어디서나 자신의 게임 데이터를 안전하게 유지할 수 있게 되었습니다. 또한, 서버 관리 부담이 줄어들어 개발 효율성이 향상되었습니다.

### 3) Dotween 기반 UI 애니메이션 시스템

- 기능 설명: UI 요소들의 등장, 퇴장, 강조 등의 애니메이션을 손쉽게 구현할 수 있는 시스템입니다.
- 구현 내용: 일부 UI 요소에 Dotween을 적용하여, 애니메이션 효과를 추가해 사용자 경험을 향상시켰습니다. 레벨업, 스테이지 클리어 등 주요 이벤트 시 애니메이션을 통해 시각적 피드백을 제공했습니다.
- 선택 이유: Unity의 기본 애니메이션 시스템보다 Dotween이 더 직관적이고 사용하기 쉬워, 빠르게 UI 애니메이션을 구현할 수 있었습니다. 또한, 코드 기반으로 애니메이션을 제어할 수 있어 유지보수에도 용이했습니다.
- 결과: UI 애니메이션이 추가되어 게임의 시각적 완성도가 높아졌으며, 사용자 경험이 향상되었습니다. 또한 코드의 재사용성이 증가하여, 향후 UI 요소 추가 시에도 쉽게 애니메이션을 적용할 수 있게 되었습니다.
